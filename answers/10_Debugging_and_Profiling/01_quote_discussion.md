# Topic

Discuss Donald Knuth’s original quote about premature
optimization. What is your opinion?

# Opinion

Donald Knuths is absolutily right with his observation. With the introduction of scrum in modern software development all of the mentioned things even got worse. 

Scrum indicates that software developer need to write 100% correct code all the time because there is no time or a special sprint for refactoring or cleaning up the code. Developers are expected to produce the perfect, optimized software right from the star, while also picking the correct architecture model for the task. 

Even though if one picks a architecture for the code right from the start of the project the project is doomed. People will use approximalty 80% of their energy in writing boiler plate code that doesn't do anything yet but to stick with the architecture instead of making an minimal viable product and see if the project is even worth it to put more time into it.

This could also explain why the percentage number of projects that fail because of software is rising.

His last statement about the compilers should tell the programmmer which part is used the most could be wrong. My first intuition is that i would want the compiler to be as lightweight as possbile but yet i have to little experience to have a say if this is a good idea or not.
